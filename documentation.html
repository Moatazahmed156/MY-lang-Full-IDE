⬡ Compiler Pipeline

From code to output

Five sequential stages transform your MY-Lang source code into executable output.

01
Lexer — Tokenization
Reads source characters and produces a flat list of typed tokens. Handles all MY-Lang token types: KEYWORD, IDENTIFIER, NUMBER, STRING, BOOLEAN, OPERATOR, PUNCTUATION. Supports escape sequences, single and multi-line comments, and error-recovery for unknown characters.
lexer.js
↓
02
Parser — AST Construction
Recursive descent parser that builds a typed AST from the token stream. Handles full expression precedence (logical → comparison → additive → multiplicative → unary → primary), all statement types, block scoping, and error recovery by continuing parse after syntax mistakes.
parser.js
↓
03
Semantic Analysis — Validation
Multi-scope semantic analyzer that walks the AST. Maintains a Scope chain for variable resolution across blocks. Builds the symbol table, detects undeclared variables, duplicate declarations, type mismatches (errors for hard conflicts, warnings for coercions), and validates array operations.
semantic.js
↓
04
Code Generation — Pseudo-Assembly
AST visitor that generates readable pseudo-assembly with .DATA and .TEXT sections. Allocates named temporary registers (t1, t2...), generates jump labels for if/else and while constructs, emits inline comments for readability, and handles all expression and statement node types.
codegen.js
↓
05
Execution — Tree-Walking Interpreter
Direct AST interpreter with a dynamic Environment for variable storage. Supports all control flow, array index reads/writes, 13 built-in functions (print, len, sqrt, abs, floor, ceil, round, max, min, pow, str, num, random), runtime error messages, and a 10,000-iteration infinite-loop guard.
executor.js
